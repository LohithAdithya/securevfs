About the Project

SecureFS v2 is a modern, enterprise-grade file encryption system built on AES-256-GCM. It features identity-bound encryption, an immutable blockchain audit ledger, and intrusion detection—all operating seamlessly in your browser or through our Python backend.

🔒
AES-256-GCM Encryption
Military-grade authenticated encryption. Files are cryptographically bound to your identity, ensuring only authorized personnel can decrypt them.
👥
Role-Based Access
Six distinct roles from Auditor to Admin. Granular permissions dictate who can encrypt, decrypt, verify, or manage users across the system.
⛓
Blockchain Audit Ledger
Every action is recorded in an immutable, cryptographically linked chain of blocks. Any tampering attempt is mathematically detectable.
🛡
Intrusion Detection
Active monitoring automatically locks user accounts after multiple failed authentication attempts, securing your data against brute-force attacks.

How It Works

SecureFS integrates advanced encryption, identity management, and blockchain auditing into a single seamless flow.

1 Identity-Bound Encryption
👤
User Identity
Username and Employee ID form the Additional Authenticated Data (AAD), uniquely binding the file to you.
🔑
Key Derivation
A secure Session Key is derived using PBKDF2-SHA256 with 100,000 iterations to resist brute-force attacks.
🛡
AES-256-GCM
The file is encrypted with military-grade AES-256-GCM. Any modification instantly breaks the integrity tag.
2 Immutable Audit Trail
⚡
Action Triggered
A sensitive operation such as Encrypt, Decrypt, or Verify is performed by a logged-in user.
🔗
Hash Linking
The action metadata is combined and hashed alongside the previous block's SHA-256 hash.
⛓
New Block Mined
An immutable ledger record is created. Retroactive tampering mathematically breaks the entire chain.
